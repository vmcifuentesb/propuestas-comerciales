1. Resumen Ejecutivo y Diagnóstico del Proyecto

Esta propuesta detalla la estrategia de desarrollo e ingeniería para el nuevo Módulo de Inventario a la medida de Keystone.

La solución propuesta está específicamente diseñada para mitigar las fallas operativas y la pérdida de control que conllevan los procesos basados en papel y plantillas de Excel dispersas.

Mediante una arquitectura nativa omnicanal y un enfoque híbrido, el proyecto está pensado para reducir al mínimo la resistencia operativa inicial de las agencias. Se garantiza esto mediante el diseño de interfaces de usuario idénticas a los requerimientos aprobados y, posteriormente, una transición segura y estructurada hacia un modelo SaaS (Software as a Service) de membresía mensual con infraestructura de nube dedicada y escalable.

Pilares Fundamentales

Arquitectura Omnicanal

Operable en PC de escritorio, tablets y dispositivos móviles de bodega.

Soberanía de Datos (BYOI)

Infraestructura dedicada 100% propiedad del cliente, sin contratos de dependencia.

Resistencia Cero

Interfaces adaptadas al flujo diario aprobado por sus propios operadores.

2. Alcance Funcional del Módulo de Inventario

El alcance funcional de este sistema cubre exclusivamente el flujo logístico de almacenamiento y traslados transaccionales de bodegas principales y locales de agencias:

Tablero General e Indicadores

Alertas visuales inmediatas y en tiempo real en la pantalla de inicio (destacadas en color rojo) cuando el nivel de stock de cualquier insumo caiga a un nivel crítico: menor o igual al 10% del stock mínimo definido.

Crítico

Carga Omnicanal Física

Entradas versátiles de mercadería: digitalización manual directa, escaneo híbrido de códigos QR y barras (usando la cámara del dispositivo móvil con Capacitor o pistolas USB en computadoras) e importación de plantillas masivas de Excel con validación de fila exacta del error.

Eficiencia

Doble Factor de Traslado

Control interagencias estricto. Al enviar insumos, se descuentan del stock de la Bodega Central en estado "En Tránsito". El inventario de la sede de destino no se incrementará hasta que el operador local use la Pantalla de Aceptación de Ingreso y verifique la carga física.

Seguridad

Segmentación Histórica

Optimización y velocidad de base de datos para manejar hasta 400,000 boletas de inventario trimestrales. Por defecto, la pantalla principal renderiza el trimestre activo, derivando consultas antiguas a una pestaña de Históricos con filtros optimizados.

Rendimiento

3. Arquitectura y Stack Tecnológico

La pila de desarrollo se estructurará bajo un ecosistema moderno unificado en TypeScript (API-First), garantizando fluidez transaccional y máximo control de tipos:

Frontend

React.js (v19), Vite para empaquetado ultra rápido, TypeScript, Tailwind CSS (v4) y componentización shadcn/ui. Portabilidad nativa con Capacitor.

Backend

NestJS (v10) en TypeScript. Alertas bidireccionales en tiempo real mediante Socket.io. Autenticación y seguridad Passport.js reforzada con JWT.

Persistencia

PostgreSQL optimizada como motor transaccional relacional central y Prisma ORM (v5.8) para el control integral de tipos seguros de base de datos.

Infraestructura

Vercel/Netlify Premium CDN para cliente web, AWS AppRunner o Render Dedicado para la API y AWS S3 / Cloudflare R2 para almacenamiento de documentos.

7. Blindaje Contractual y Garantías UAT

El proceso de aseguramiento de calidad y control contractual se regirá estrictamente bajo las siguientes cláusulas del protocolo de Keystone:

Garantía de Aceptación UAT

Las pruebas en Staging se limitarán a una ventana de 5 días hábiles guiada por dos usuarios clave de Keystone (Administrador Global y Bodeguero Central). Se validará el login JWT, importación de Excel con errores, escáner en bodegas y el doble factor de traslado.

Control de Cambios Estricto

Cualquier funcionalidad no descrita en esta propuesta o la reactivación del módulo de compras se procesará como un requerimiento adicional (Change Request) bajo una tarifa fija de Q200.00 por hora hombre de ingeniería.

Suspensión por Impago

El retraso de más de 5 días hábiles en los pagos de licenciamiento mensual faculta a la suspensión inmediata del entorno cloud de producción (AWS/Render), bloqueando los accesos hasta la regularización de saldos, sin responsabilidad por interrupción de negocio.

Congelamiento de Código (Feature Freeze)

Al desplegar en Staging (Hito 3), el código se congelará por completo. Durante este periodo únicamente se corregirán errores (bugs) sobre lo programado. Las iteraciones de diseño o nuevos flujos se pospondrán para una futura versión v2.1.
